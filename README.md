# ai-timer-worked

Локальный трекер рабочего времени: трей-таймер (`timer/`) пишет события в
PostgreSQL, веб-дашборд (`dashboard/`) читает их и рисует отчёты в браузере.
Один установщик поднимает всё окружение целиком.

## Установка

```bash
bash install.sh
```

Скрипт (нужен `sudo`, интерактивные запросы пароля возможны):

1. Ставит системные зависимости таймера, локальный PostgreSQL и Node.js 22.
2. Создаёт роль и базу данных PostgreSQL, накатывает миграции из `sql/migrations/`.
3. Готовит venv таймера (`timer/.venv`) и собирает продакшен-сборку дашборда.
4. Прописывает `timer/.env` и `dashboard/.env` с общей строкой подключения.
5. Регистрирует автозапуск: трей-таймер — через `~/.config/autostart`,
   дашборд — как systemd `--user` сервис `ai-work-hours-dashboard`
   на `http://localhost:3000`.

После установки дашборд открывается и из трей-меню таймера: **🌐 Открыть дашборд**.

## Структура

```
ai-timer-worked/
├── install.sh          # единый установщик
├── sql/
│   ├── migrations/      # общие миграции (см. корневой CLAUDE.md — конвенция именования)
│   └── STRUCTURE.md      # текущая структура БД
├── timer/                # трей-таймер (Python/GTK, Linux)
└── dashboard/             # веб-дашборд (Next.js)
```

Подробности по каждой части — в `timer/README.md` / `timer/GUIDE.md` и `dashboard/README.md`.
