# Work Timer

Трекер рабочего времени для Linux Mint. Живёт в системном трее — зелёный кружок в ожидании, красный во время работы. Сессии записываются в Supabase или в любой PostgreSQL, отчёты выгружаются в Markdown.

> Это часть моно-репозитория `ai-timer-worked`. Для установки готового проекта
> целиком (локальный PostgreSQL + миграции + этот таймер + веб-дашборд)
> используйте `../install.sh` из корня репозитория, а не шаги ниже.

## Возможности

- **Старт / пауза / стоп** — ввод названия задачи через диалог, логирование в БД; пауза не засчитывается в затраченное время
- **Supabase или PostgreSQL** — при первом запуске мастер настройки спрашивает, каким способом подключаться к БД
- **Автоматические миграции** — при старте приложение само применяет ещё не применённые файлы из `sql/migrations`, если задана строка подключения PostgreSQL
- **Статистика за сегодня** — таблица сессий с временем и итогом
- **Отчёт за неделю и месяц** — сохранение `.md`-файла с разбивкой по дням, запоминает последнюю папку сохранения
- **Автозапуск** — скрипт создаёт `~/.config/autostart/work-timer.desktop`

## Стек

Python 3.12 · pystray (ayatana-appindicator) · Pillow · supabase-py · psycopg2 · python-dotenv · zenity · notify-send · wmctrl

## Быстрый старт

```bash
# 1. Системные зависимости
sudo apt install python3-gi python3-gi-cairo \
    gir1.2-ayatanaappindicator3-0.1 libayatana-appindicator3-1 \
    libnotify-bin zenity wmctrl

# 2. Виртуальное окружение
/usr/bin/python3 -m venv .venv --system-site-packages
.venv/bin/pip install -r requirements.txt

# 3. Запуск
.venv/bin/python timer.py
```

При первом запуске появится мастер настройки: выбор Supabase или PostgreSQL и ввод реквизитов подключения — они сохранятся в `.env`. Если указана строка подключения PostgreSQL (`DATABASE_URL`, для Supabase — необязательно, для прямого PostgreSQL — обязательно), при каждом запуске приложение само проверяет и применяет ещё не применённые миграции из общего каталога `../sql/migrations`.

Без мастера — заполните `.env` вручную (см. `.env.example`) и выполните `../sql/migrations/*.sql` по порядку в Supabase SQL Editor / `psql`.

## Структура таблицы

Таблица `wh_work_log`. Каждая сессия — цепочка строк `start` → (`pause` / `resume`)* → `stop`, связанных по `session_id`.

| Поле | Тип | Описание |
|------|-----|----------|
| `session_id` | uuid | Связывает все события одной сессии |
| `operation` | varchar(6) | `start`, `pause`, `resume` или `stop` |
| `task` | text | Название задачи |
| `event_time` | timestamptz | Время события (UTC) |
| `elapsed_time` | text | Затраченное время без пауз (только у `stop`) |

## Безопасность

Проведён статический анализ кода. Уязвимостей не обнаружено.

| Область | Результат | Примечание |
|---------|-----------|------------|
| Инъекция команд | ✅ Безопасно | Все вызовы `subprocess` используют список аргументов, `shell=False` |
| SQL-инъекция | ✅ Безопасно | Supabase — параметризованный API (`.eq()`, `.gte()`, `.in_()`); PostgreSQL-бэкенд — параметризованные запросы psycopg2 (`%s`), имена таблиц не зависят от пользовательского ввода |
| Временные файлы | ✅ Безопасно | `NamedTemporaryFile` + удаление в `finally` |
| Пользовательский ввод | ✅ Безопасно | Текст задачи экранируется при записи в Markdown (`\|`) |
| Секреты | ✅ Безопасно | Credentials только через переменные окружения, `.env` исключён из git |
